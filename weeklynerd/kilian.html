Meenemen

Kilian Valkhof maakt dingen via Electron en heeft de Polypane browser gebouwd. Wat hij vertelde vond ik eigenlijk heel logisch, maar ik had er nog nooit zo over nagedacht. Hij had het over "the rule of least power": als je iets gaat bouwen, moet je kijken of je het in de minst krachtige taal kan bouwen. Dus eerst HTML, dan CSS, en pas als het echt niet anders kan JavaScript. Dat klinkt simpel, maar ik merk dat ik zelf al snel naar JavaScript grijp terwijl het eigenlijk ook in CSS kan. Dat is iets wat ik echt mee ga nemen. Ook het idee dat je na een paar maanden moet checken of er een betere manier is om iets te bouwen vond ik fijn, het voelt als een soort reminder om niet lui te worden in hoe je code schrijft.

Oefenen

Wat ik zeker wil oefenen is het terugkijken naar mijn eigen code en kijken wat ik kan vereenvoudigen. Minder JavaScript, meer CSS. Dat voelt als een uitdaging maar ook gewoon als de juiste manier van werken. Dit probeer ik al in deze website te doen.
